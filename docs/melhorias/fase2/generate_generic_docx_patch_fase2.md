# generate_generic_docx.py — fase 2 de patch

## Objetivo
Detalhar a conversão para DOCX com foco em consistência e revisão.

## Regras de formatação
- Times New Roman ou fonte equivalente do template.
- Título destacado.
- Seções com cabeçalhos claros.
- Parágrafos curtos.
- Tabelas apenas quando agregarem leitura.

## Checklist de alteração
- Manter mesma ordem do relatório em Markdown.
- Preservar resumo, processamento, evidências e conclusão.
- Evitar estilos complexos que dificultem manutenção.
- Garantir que o arquivo seja fácil de revisar localmente.

## Observações
- O DOCX é uma exportação, não uma nova interpretação.
- O conteúdo deve continuar o mesmo do relatório-base.
